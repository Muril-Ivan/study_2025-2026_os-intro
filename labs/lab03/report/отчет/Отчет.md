## Контрольные вопросы
1. Что такое системы контроля версий (VCS) и для решения каких задач они предназначаются?
 Это инструменты для отслеживания изменений в проекте. Они предназначены для управления историей изменений,восстановления и предотвращения потери данных.
2. Объясните следующие понятия VCS и их отношения: хранилище, commit, история, рабочая копия.
    1. Хранилище - это место, где хранится данные и реквизиты проекта.
    2. Commit - это комментарий к изменениям вносимым в проект.
    3. История - это последовательность всех коммитов.То есть история измениния.
    4. Рабочия копия - текущая версия проекта.
3. Что представляют собой и чем отличаются централизованные и децентрализованные VCS? Приведите примеры VCS каждого вида.
    1. Централизованные VCS (например, SVN) имеют одно центральное хранилище, куда все пользователи коммитят изменения; разработчики берут только часть кода и отправляют обратно в центральное хранилище.
    2. Децентрализованные VCS (например, Git) позволяют каждому иметь полную копию всего репозитория, включая всю историю, и изменения можно обмениваться напрямую между участниками.
    3. Основное отличие:Централизованные требуют постоянного соединения с сервером, а децентрализованные работают автономно и позволяют более гибкое управление.
      1.Примеры:  
        1. Централизованные: SVN, CVS  
        2. Децентрализованные: Git, Mercurial

6. Каковы основные задачи, решаемые инструментальным средством git?
    Сохраниение данных и удобство редакции, а также обмен ими. 
7. Назовите и дайте краткую характеристику командам git.
    1. git clone — копирует удалённое хранилище к себе на компьютер.
    2. git status — отображает состояние рабочей копии и индекс.
    3. git add — добавляет файлы или изменения в индекс для следующего коммита.
    4. git commit — сохраняет зафиксированные изменения в локальной истории.
    5. git pull — получает последние изменения из удалённого репозитория и объединяет их с локальными.
    6. git push — отправляет локальные коммиты в удалённое хранилище.

10. Как и зачем можно игнорировать некоторые файлы при commit 
В git для игнорирования некоторых файлов при коммите используют файл .gitignore. Он содержит список шаблонов файлов и папок, которые Git должен игнорировать — то есть не добавлять и не отслеживать их изменения
